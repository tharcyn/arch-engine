# Arch-Engine v1.0.0

## What Arch-Engine is

Arch-Engine is infrastructure for repository architecture reasoning.
It extracts repository topology directly from source structure
and produces deterministic closureGraphHash-stable topology graphs.
Composable policy packs optionally evaluate architectural invariants
over that topology.

## What Arch-Engine is not

Arch-Engine is not:
- a linter
- a build system
- a static rule engine
- a tracing framework
- a CI replacement
It operates alongside existing tooling.

## Why topology identity matters

Most architecture tooling evaluates rules.
Arch-Engine evaluates topology identity first.
Deterministic topology identity enables:
- stable snapshot comparison
- structural drift detection
- policy-pack federation
- capability negotiation
- overlay-safe governance composition

## Quickstart

```bash
npm install @arch-engine/cli
npx arch-engine doctor
```

No configuration required.
Runs fully offline by default.
Creates a local `.arch-engine/` context directory on first execution only.

## Policy-pack optionality

Policy packs extend the runtime with invariant evaluation.
The diagnostic runtime executes safely without them.
They can be installed later without changing extraction behavior.

## Deterministic snapshot model

Arch-Engine produces closureGraphHash-stable topology graphs.
Snapshot envelopes remain transport-stable across executions
within identical repository states.

## Offline execution guarantee

Arch-Engine performs no network requests by default.
Federation features activate only when explicitly configured.
