# Arch-Engine v1.0.0-rc.3

Release candidate 3 finalizes the deterministic runtime constellation and separates core topology intelligence from ambient host execution vectors.

## Runtime Freeze

The following foundational packages are **frozen** at `1.0.0-rc.3`. Their public API surfaces, export maps, and execution semantics are locked and will not change before `1.0.0` stable.

| Package | Version | Surface |
| --- | --- | --- |
| `@arch-engine/schema` | `1.0.0-rc.3` | Canonical types, JSON schema contracts |
| `@arch-engine/core` | `1.0.0-rc.3` | Engine runner, manifest loader, policy system, analysis, parsers |
| `@arch-engine/cli` | `1.0.0-rc.3` | `doctor`, `inspect`, `check`, `analyze`, `explain` commands |

### Frozen guarantees

- `closureGraphHash` identity stability
- Snapshot replay compatibility
- Policy-pack participation grammar
- Registry federation transport neutrality
- Diagnostic schema (`R0-v1`) output contract

## Extension Refinement

The following optional extensions are versioned at `1.0.0-rc.4`, reflecting continued refinement independent of the frozen core surface. These packages are **not bundled** with core ‚Äî they are installed separately.

| Package | Version | Purpose |
| --- | --- | --- |
| `@arch-engine/adapter-monorepo` | `1.0.0-rc.4` | Workspace topology extraction (npm, yarn, pnpm) |
| `@arch-engine/governance-pack-authority` | `1.0.0-rc.4` | Authority boundary and blast radius governance |
| `@arch-engine/governance-pack-rest-contract` | `1.0.0-rc.4` | REST contract parity validation (operationId, annotations) |
| `@arch-engine/governance-pack-journey` | `1.0.0-rc.4` | Journey lifecycle coverage regression detection |

### Extension dependency contract

All extensions depend on `@arch-engine/core@1.0.0-rc.3`. The adapter and governance packs do **not** depend on each other and can be installed independently.

## Upgrade Notes

If upgrading from `rc.1` or `rc.2`:

```bash
npm install @arch-engine/core@1.0.0-rc.3 @arch-engine/schema@1.0.0-rc.3 @arch-engine/cli@1.0.0-rc.3
```

Optional extensions can be upgraded independently:

```bash
npm install @arch-engine/adapter-monorepo@1.0.0-rc.4
npm install @arch-engine/governance-pack-authority@1.0.0-rc.4
```
